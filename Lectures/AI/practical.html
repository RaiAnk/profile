Welcome to Prolog Programming!

Don't worry if you've never programmed before! Prolog is actually one of the easiest languages to understand because it works like how we think naturally - by asking questions and defining relationships.

What is Prolog?

Prolog = Programming in Logic

Instead of telling the computer HOW to solve a problem step-by-step, you tell it WHAT the problem is, and it figures out the solution!

Think of it as describing facts and relationships, then asking questions.

Why Learn Prolog?

  • Perfect for AI and expert systems
  • Natural language processing
  • Database querying
  • Solving puzzles and games
  • Teaches logical thinking

How Does It Work?

Three simple building blocks:

  1. Facts - Things that are true
  2. Rules - IF-THEN relationships
  3. Queries - Questions you ask

Where is it Used?

  • IBM Watson (AI assistant)
  • Expert medical diagnosis systems
  • Natural language chatbots
  • Scheduling and planning systems

Prolog Basics - Learn in 5 Minutes!

Think of Prolog Like a Detective!

Imagine you're a detective. You have:

  • Facts = Evidence you know is true ("John is a man", "Mary is John's mother")
  • Rules = Logical deductions ("If X is Y's mother, then Y is X's child")
  • Queries = Questions to solve ("Who is John's mother?")

Essential Prolog Syntax

fact. Every statement ends with a period (.)
likes(john, pizza). Predicates describe relationships. Here: "John likes pizza"
X, Y, Name Variables start with UPPERCASE letters
john, pizza, mother Constants/Atoms start with lowercase letters
parent(X, Y) :- father(X, Y). Rules use :- which means "if". Read as: "X is parent of Y IF X is father of Y"
?- likes(john, pizza). Queries start with ?- to ask questions
A, B AND operator: comma means "and"
A ; B OR operator: semicolon means "or"

Real-Life Analogy

Imagine a family tree database:

% Facts (things we know)

father(ram, sita).

% This simply means: "Ram is the father of Sita"

mother(geeta, sita).

% This means: "Geeta is the mother of Sita"

Now you can ask questions like: ?- father(X, sita). and Prolog will tell you X = ram

Practical 1

Family Relationships using Facts and Rules

CO1: Represent Production Rules

Aim

Write a Prolog program to find the rules for parent, child, son, daughter, brother, sister, uncle, aunt, ancestor given only facts about man, woman, father, and wife.

Understanding the Problem

Imagine you have a family tree. We'll give Prolog just 4 basic facts:

  • man(X) - X is a male person
  • woman(X) - X is a female person
  • father(X, Y) - X is the father of Y
  • wife(X, Y) - X is the wife of Y

From these 4 facts, Prolog can figure out ALL other relationships through logical rules!

The Logic Behind Family Rules

Relationship Logic in English
Mother If X is wife of Y, and Y is father of Z, then X is mother of Z
Parent If X is father OR mother of Y, then X is parent of Y
Child If X is parent of Y, then Y is child of X
Son If X is child of Y AND X is a man, then X is son of Y
Daughter If X is child of Y AND X is a woman, then X is daughter of Y
Brother If X and Y have same parent AND X is a man AND X is not Y
Sister If X and Y have same parent AND X is a woman AND X is not Y
Uncle If X is brother of Y AND Y is parent of Z, then X is uncle of Z
Aunt If X is sister of Y AND Y is parent of Z, then X is aunt of Z
Ancestor X is ancestor of Y if X is parent of Y, OR X is parent of someone who is ancestor of Y (recursive!)

Complete Program

family.pl 
% ============================================
% FACTS: Basic information about the family
% ============================================

% Defining who are men
man(ram).
man(shyam).
man(mohan).
man(sohan).
man(rahul).

% Defining who are women
woman(sita).
woman(gita).
woman(radha).
woman(meera).

% Defining father relationships
% father(X, Y) means X is the father of Y
father(ram, mohan).
father(ram, gita).
father(shyam, rahul).
father(shyam, meera).
father(mohan, sohan).
father(mohan, radha).

% Defining wife relationships
% wife(X, Y) means X is the wife of Y
wife(sita, ram).
wife(gita, shyam).

% ============================================
% RULES: Derived relationships
% ============================================

% Mother: X is mother of Y if X is wife of someone who is father of Y
mother(X, Y) :-
    wife(X, Z),
    father(Z, Y).

% Parent: X is parent of Y if X is father OR mother of Y
parent(X, Y) :- father(X, Y).
parent(X, Y) :- mother(X, Y).

% Child: Y is child of X if X is parent of Y
child(Y, X) :- parent(X, Y).

% Son: X is son of Y if X is child of Y and X is a man
son(X, Y) :-
    child(X, Y),
    man(X).

% Daughter: X is daughter of Y if X is child of Y and X is a woman
daughter(X, Y) :-
    child(X, Y),
    woman(X).

% Sibling: X and Y are siblings if they have the same parent
sibling(X, Y) :-
    parent(Z, X),
    parent(Z, Y),
    X \= Y.

% Brother: X is brother of Y if they are siblings and X is a man
brother(X, Y) :-
    sibling(X, Y),
    man(X).

% Sister: X is sister of Y if they are siblings and X is a woman
sister(X, Y) :-
    sibling(X, Y),
    woman(X).

% Uncle: X is uncle of Y if X is brother of Y's parent
uncle(X, Y) :-
    brother(X, Z),
    parent(Z, Y).

% Aunt: X is aunt of Y if X is sister of Y's parent
aunt(X, Y) :-
    sister(X, Z),
    parent(Z, Y).

% Ancestor: X is ancestor of Y if X is parent of Y (base case)
ancestor(X, Y) :- parent(X, Y).

% Ancestor: X is ancestor of Y if X is parent of someone who is ancestor of Y (recursive)
ancestor(X, Y) :-
    parent(X, Z),
    ancestor(Z, Y).

Running Queries

Practical 2

Implementation of Cut (!) and Fail Predicates

CO2: Control Backtracking

Aim

Understand and implement the cut (!) and fail predicates to control Prolog's backtracking behavior.

What is Backtracking?

When Prolog tries to solve a query, it explores all possible solutions. If one path fails, it "backtracks" - goes back and tries another path.

Think of it like GPS navigation: If one road is blocked, GPS finds an alternative route. Prolog does the same with logic!

Understanding Cut (!)

The cut (!) is like a "no going back" sign.

Once Prolog passes a cut:

  • It commits to all choices made so far in this rule
  • It won't backtrack to try other alternatives
  • It's like saying "I'm satisfied with this answer, don't look for more"

Analogy: Finding a Restaurant

Without cut: "Find ALL good restaurants in town" - keeps searching even after finding one

With cut: "Find the FIRST good restaurant and STOP" - once found, stops looking

Example 1: Without Cut (Multiple Solutions)

without_cut.pl 
% Facts about fruits
fruit(apple).
fruit(banana).
fruit(cherry).
fruit(date).

% Rule to find any fruit
find_fruit(X) :-
    fruit(X),
    write('Found: '),
    write(X),
    nl.
Output (Without Cut)
?- find_fruit(X).
Found: apple
X = apple ;
Found: banana
X = banana ;
Found: cherry
X = cherry ;
Found: date
X = date.

Example 2: With Cut (Stop at First)

with_cut.pl 
% Facts about fruits
fruit(apple).
fruit(banana).
fruit(cherry).

% Rule with cut - finds only the first fruit and stops
find_first_fruit(X) :-
    fruit(X),
    !,  % Cut here - don't backtrack!
    write('Found first fruit: '),
    write(X),
    nl.
Output (With Cut)
?- find_first_fruit(X).
Found first fruit: apple
X = apple.

% Notice: Only ONE solution, no more backtracking!

Understanding Fail

The fail predicate always fails.

It forces Prolog to backtrack and try other solutions. Common uses:

  • Combined with repeat to create loops
  • To print all solutions
  • Combined with cut to create negation

Example 3: Using Fail to Print All Solutions

print_all.pl 
% Facts
student(alice).
student(bob).
student(charlie).
student(diana).

% Print all students using fail
print_all_students :-
    student(X),
    write(X), nl,
    fail.  % Force backtracking to find next solution

% This second clause succeeds when all solutions are exhausted
print_all_students.
Output
?- print_all_students.
alice
bob
charlie
diana
true.

Example 4: Cut-Fail Combination (Negation)

negation.pl 
% Facts about likes
likes(mary, pizza).
likes(mary, ice_cream).
likes(john, burger).

% Negation as failure: X does NOT like Y
does_not_like(X, Y) :-
    likes(X, Y),
    !,         % If likes(X,Y) succeeds, cut
    fail.      % Then fail - so does_not_like fails
does_not_like(_, _).  % Otherwise, succeed (X does not like Y)

% Practical example: Find students who are NOT on the honor roll
honor_roll(alice).
honor_roll(charlie).

regular_student(X) :-
    student(X),
    \+ honor_roll(X).  % \+ is built-in negation
Output
?- does_not_like(mary, pizza).
false.

?- does_not_like(mary, burger).
true.

?- regular_student(X).
X = bob ;
X = diana.

Practical 3

Calculate Factorial Value

CO2: Control Backtracking

Aim

Write a Prolog program to calculate the factorial of a given number using recursion.

What is Factorial?

Factorial of N (written as N!) = N (N-1) (N-2) ... 2 1

  • 5! = 5 4 3 2 1 = 120
  • 4! = 4 3 2 1 = 24
  • 3! = 3 2 1 = 6
  • 0! = 1 (by definition)

Recursive Definition

Factorial can be defined recursively:

  • Base case: factorial(0) = 1
  • Recursive case: factorial(N) = N factorial(N-1)

Example: factorial(4) = 4 factorial(3) = 4 3 factorial(2) = 4 3 2 factorial(1) = 4 3 2 1 factorial(0) = 4 3 2 1 1 = 24

Complete Program

factorial.pl 
% Factorial calculation in Prolog
% factorial(N, F) means F is the factorial of N

% Base case: factorial of 0 is 1
factorial(0, 1) :- !.

% Recursive case: factorial of N is N * factorial(N-1)
factorial(N, F) :-
    N > 0,                    % N must be positive
    N1 is N - 1,              % Calculate N-1
    factorial(N1, F1),         % Find factorial of N-1
    F is N * F1.               % F = N * factorial(N-1)

% User-friendly version with output
calculate_factorial(N) :-
    factorial(N, F),
    write('Factorial of '), write(N),
    write(' is '), write(F), nl.
Sample Output
?- factorial(5, F).
F = 120.

?- factorial(0, F).
F = 1.

?- factorial(7, F).
F = 5040.

?- calculate_factorial(6).
Factorial of 6 is 720
true.

How It Works: Step by Step

  1. Query: factorial(4, F) - Find factorial of 4
  2. 4 > 0, so calculate N1 = 4-1 = 3, need factorial(3, F1)
  3. 3 > 0, so calculate N1 = 3-1 = 2, need factorial(2, F1)
  4. 2 > 0, so calculate N1 = 2-1 = 1, need factorial(1, F1)
  5. 1 > 0, so calculate N1 = 1-1 = 0, need factorial(0, F1)
  6. Base case: factorial(0, 1) - F1 = 1
  7. Now unwind: F = 1*1 = 1, F = 2*1 = 2, F = 3*2 = 6, F = 4*6 = 24

Experiments 4-8: Working with Lists

What is a List in Prolog?

A list is a collection of elements enclosed in square brackets, separated by commas.

  • [1, 2, 3, 4, 5] - A list of numbers
  • [apple, banana, cherry] - A list of atoms
  • [] - An empty list
  • [Head | Tail] - Head is first element, Tail is rest of the list

The Head-Tail Pattern (Most Important!)

Every non-empty list can be split into:

  • Head - The first element
  • Tail - Everything else (also a list)

?- [H|T] = [1, 2, 3, 4, 5].

H = 1

T = [2, 3, 4, 5]

display_list.pl 
% Display all elements in a list

% Base case: Empty list - nothing to print
print_list([]).

% Recursive case: Print head, then print tail
print_list([H|T]) :-
    write(H), nl,      % Print the head element
    print_list(T).      % Recursively print the tail

% Alternative: Print with numbering
print_numbered(List) :- print_numbered(List, 1).

print_numbered([], _).
print_numbered([H|T], N) :-
    write(N), write('. '), write(H), nl,
    N1 is N + 1,
    print_numbered(T, N1).
Output

banana
cherry
date
true.

2. green
3. blue
true.

Find Length of a List

Aim

list_length.pl 
% Find length of a list
% list_length(List, Length)

% Base case: Empty list has length 0
list_length([], 0).

% Recursive case: Length = 1 + length of tail
list_length([_|T], Len) :-
    list_length(T, TailLen),
    Len is TailLen + 1.

% Tail-recursive version (more efficient)
list_length_tr(List, Len) :- list_length_tr(List, 0, Len).

list_length_tr([], Acc, Acc).
list_length_tr([_|T], Acc, Len) :-
    Acc1 is Acc + 1,
    list_length_tr(T, Acc1, Len).

Merge Two Lists

CO3: Implement List Operations

Aim

Write a Prolog program to merge (append/concatenate) two lists into one.

Understanding Merge

Merging [1,2,3] and [4,5,6] gives [1,2,3,4,5,6]

Logic: Take elements from first list one by one, prepend to the merged result.

merge_list.pl 
% Merge two lists (also called append)
% merge(List1, List2, MergedList)

% Base case: Merging empty list with L gives L
merge([], L, L).

% Recursive case: Take head of first list, merge rest
merge([H|T1], L2, [H|T3]) :-
    merge(T1, L2, T3).

% Display merged result
show_merge(L1, L2) :-
    merge(L1, L2, Result),
    write('List 1: '), write(L1), nl,
    write('List 2: '), write(L2), nl,
    write('Merged: '), write(Result), nl.

% Merge three lists (bonus)
merge3(L1, L2, L3, Result) :-
    merge(L1, L2, Temp),
    merge(Temp, L3, Result).
Output

Delete an Item from List

CO3: Implement List Operations

Aim

Write a Prolog program to delete an item from a list.

delete_item.pl 
% Delete an item from a list
% delete_item(Item, List, ResultList)

% Base case: Item not in empty list
delete_item(_, [], []).

% If head matches item, skip it (delete it)
delete_item(X, [X|T], T) :- !.  % Delete first occurrence only

% If head doesn't match, keep it and continue
delete_item(X, [H|T], [H|R]) :-
    delete_item(X, T, R).

% Delete ALL occurrences of an item
delete_all(_, [], []).
delete_all(X, [X|T], R) :-
    delete_all(X, T, R).  % Skip X and continue
delete_all(X, [H|T], [H|R]) :-
    X \= H,
    delete_all(X, T, R).

% Check if item is a member of the list
is_member(X, [X|_]) :- !.
is_member(X, [_|T]) :- is_member(X, T).
Output
?- delete_item(3, [1, 2, 3, 4, 5], R).
R = [1, 2, 4, 5].

?- delete_item(a, [a, b, a, c, a], R).
R = [b, a, c, a].    % Only first 'a' deleted

?- delete_all(a, [a, b, a, c, a], R).
R = [b, c].          % All 'a's deleted

?- is_member(3, [1, 2, 3, 4]).
true.

?- is_member(5, [1, 2, 3]).
false.

Practical 9

Tower of Hanoi Problem

CO4: Solve Knowledge-Based Problems

Aim

Write a Prolog program to solve the classic Tower of Hanoi puzzle.

What is Tower of Hanoi?

A classic puzzle with three pegs and N disks of different sizes.

Goal: Move all disks from peg A (source) to peg C (destination).

Rules:

  • Only one disk can be moved at a time
  • Only the top disk can be moved
  • A larger disk cannot be placed on a smaller disk
Tower of Hanoi Visualization
    Initial State:          Goal State:

         |                      |
        [1]                     |
       [ 2 ]                    |
      [  3  ]                   |
    =========              =========
       A         B            C

    Move all disks from A to C using B as auxiliary

The Recursive Strategy

To move N disks from A to C using B as auxiliary:

  1. Move N-1 disks from A to B (using C as auxiliary)
  2. Move the largest disk from A to C
  3. Move N-1 disks from B to C (using A as auxiliary)
hanoi.pl 
% Tower of Hanoi Solution
% hanoi(N, Source, Auxiliary, Destination)

% Base case: Move single disk directly
hanoi(1, Source, _, Dest) :-
    write('Move disk from '),
    write(Source),
    write(' to '),
    write(Dest),
    nl.

% Recursive case: Move N disks
hanoi(N, Source, Aux, Dest) :-
    N > 1,
    N1 is N - 1,
    % Step 1: Move N-1 disks from Source to Auxiliary
    hanoi(N1, Source, Dest, Aux),
    % Step 2: Move largest disk from Source to Destination
    write('Move disk from '),
    write(Source),
    write(' to '),
    write(Dest),
    nl,
    % Step 3: Move N-1 disks from Auxiliary to Destination
    hanoi(N1, Aux, Source, Dest).

% Easy-to-use wrapper
solve_hanoi(N) :-
    write('=== Tower of Hanoi with '), write(N), write(' disks ==='), nl,

Practical 10

Water Jug Problem / Missionary Cannibal Problem

CO4: Solve Knowledge-Based Problems

Aim

Write a Prolog program to solve the Water Jug Problem.

The Water Jug Problem

Given: Two jugs - one holds 4 liters, another holds 3 liters. No measuring marks.

Goal: Measure exactly 2 liters of water.

Allowed Operations:

  • Fill a jug completely
  • Empty a jug completely
  • Pour from one jug to another until source is empty or destination is full
water_jug.pl 
% Water Jug Problem
% State is represented as state(Jug4, Jug3)
% Jug4 can hold 0-4 liters, Jug3 can hold 0-3 liters

% Capacity of jugs
capacity(jug4, 4).
capacity(jug3, 3).

% Goal: Get 2 liters in the 4-liter jug
goal(state(2, _)).

% Possible moves:

% 1. Fill Jug4 completely
move(state(_, Y), state(4, Y), 'Fill 4-liter jug').

% 2. Fill Jug3 completely
move(state(X, _), state(X, 3), 'Fill 3-liter jug').

% 3. Empty Jug4
move(state(_, Y), state(0, Y), 'Empty 4-liter jug').

% 4. Empty Jug3
move(state(X, _), state(X, 0), 'Empty 3-liter jug').

% 5. Pour from Jug3 to Jug4 (until Jug4 is full or Jug3 is empty)
move(state(X, Y), state(X1, Y1), 'Pour from 3L to 4L') :-
    Y > 0,
    Transfer is min(Y, 4 - X),
    X1 is X + Transfer,
    Y1 is Y - Transfer,
    X1 \= X.

% 6. Pour from Jug4 to Jug3 (until Jug3 is full or Jug4 is empty)
move(state(X, Y), state(X1, Y1), 'Pour from 4L to 3L') :-
    X > 0,
    Transfer is min(X, 3 - Y),
    X1 is X - Transfer,
    Y1 is Y + Transfer,
    Y1 \= Y.

% Solve using DFS with visited states
solve(State, _, []) :-
    goal(State), !.

solve(State, Visited, [Action|Actions]) :-
    move(State, NextState, Action),
    \+ member(NextState, Visited),
    solve(NextState, [NextState|Visited], Actions).

% Main solver
water_jug :-
    write('=== Water Jug Problem ==='), nl,
    write('Goal: Get 2 liters in the 4L jug'), nl, nl,
    Start = state(0, 0),
    solve(Start, [Start], Actions),
    write('Solution steps:'), nl,
    print_actions(Actions, 1).

print_actions([], _).
print_actions([H|T], N) :-
    write(N), write('. '), write(H), nl,
    N1 is N + 1,
    print_actions(T, N1).

Practical 11

SEND + MORE = MONEY (Cryptarithmetic)

CO4: Solve Knowledge-Based Problems

Aim

Write a Prolog program to solve the famous cryptarithmetic puzzle: SEND + MORE = MONEY

What is Cryptarithmetic?

A puzzle where letters represent digits (0-9). Each letter must represent a unique digit.

SEND

+ MORE

MONEY

Find: What digit does each letter represent?

Constraints: S and M cannot be 0 (no leading zeros)

cryptarithmetic.pl 
% SEND + MORE = MONEY Cryptarithmetic Puzzle
% Each letter represents a unique digit 0-9
% S and M cannot be 0 (no leading zeros)

% Main solver using CLP(FD) - Constraint Logic Programming over Finite Domains
:- use_module(library(clpfd)).

send_more_money([S,E,N,D,M,O,R,Y]) :-
    % All variables are digits 0-9
    Vars = [S,E,N,D,M,O,R,Y],
    Vars ins 0..9,

    % All digits must be different
    all_different(Vars),

    % S and M cannot be 0 (no leading zeros)
    S #\= 0,
    M #\= 0,

    % The equation: SEND + MORE = MONEY
    % SEND  = 1000*S + 100*E + 10*N + D
    % MORE  = 1000*M + 100*O + 10*R + E
    % MONEY = 10000*M + 1000*O + 100*N + 10*E + Y

    1000*S + 100*E + 10*N + D +
    1000*M + 100*O + 10*R + E #=
    10000*M + 1000*O + 100*N + 10*E + Y,

    % Find the solution
    label(Vars).

% Pretty print solution
solve :-
    write('=== SEND + MORE = MONEY ==='), nl, nl,
    send_more_money([S,E,N,D,M,O,R,Y]),

    write('Solution found!'), nl, nl,
    write('Letter = Digit mapping:'), nl,
    format('  S = ~w~n', [S]),
    format('  E = ~w~n', [E]),
    format('  N = ~w~n', [N]),
    format('  D = ~w~n', [D]),
    format('  M = ~w~n', [M]),
    format('  O = ~w~n', [O]),
    format('  R = ~w~n', [R]),
    format('  Y = ~w~n', [Y]),
    nl,

    SEND is 1000*S + 100*E + 10*N + D,
    MORE is 1000*M + 100*O + 10*R + E,
    MONEY is 10000*M + 1000*O + 100*N + 10*E + Y,

    write('Verification:'), nl,
    format('  SEND  = ~w~n', [SEND]),
    format('+ MORE  = ~w~n', [MORE]),
    write('----------'), nl,
    format('  MONEY = ~w~n', [MONEY]).
cryptarithmetic_simple.pl (Without CLP - Generate & Test) 
% Simple version without CLP library
% Uses generate-and-test approach

digit(0). digit(1). digit(2). digit(3). digit(4).
digit(5). digit(6). digit(7). digit(8). digit(9).

solve_simple(S,E,N,D,M,O,R,Y) :-
    % Generate possible values
    digit(D), digit(E), digit(Y),
    % Check column 1 (rightmost): D + E = Y or D + E = 10 + Y
    C1 is (D + E) // 10,
    Y is (D + E) mod 10,

    digit(N), digit(R),
    N \= D, N \= E, N \= Y,
    R \= D, R \= E, R \= Y, R \= N,
    % Check column 2: N + R + C1 = E or 10 + E
    C2 is (N + R + C1) // 10,
    E is (N + R + C1) mod 10,

    digit(O),
    O \= D, O \= E, O \= Y, O \= N, O \= R,
    % Check column 3: E + O + C2 = N or 10 + N
    C3 is (E + O + C2) // 10,
    N is (E + O + C2) mod 10,

    digit(S), digit(M),
    S \= 0, M \= 0,  % No leading zeros
    S \= D, S \= E, S \= Y, S \= N, S \= R, S \= O,
    M \= D, M \= E, M \= Y, M \= N, M \= R, M \= O, M \= S,
    % Check column 4: S + M + C3 = O or 10 + O
    C4 is (S + M + C3) // 10,
    O is (S + M + C3) mod 10,
    % Check column 5: C4 = M
    M is C4.

% Run the solver
run_simple :-
    solve_simple(S,E,N,D,M,O,R,Y),
    format('S=~w E=~w N=~w D=~w M=~w O=~w R=~w Y=~w~n',
           [S,E,N,D,M,O,R,Y]).
