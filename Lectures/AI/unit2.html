1. Introduction to Knowledge Representation (KR)

Knowledge Representation (KR) is the method used to encode knowledge about the world into a form that a computer system can utilize to solve complex problems. It is the study of how beliefs, intentions, and judgments of an intelligent agent can be expressed in a transparent, accessible notation for automated reasoning.

According to Davis, Shrobe & Szolovits: "A knowledge representation is fundamentally a surrogate, a substitute for the thing itself, used to enable an entity to determine consequences by thinking rather than acting."

Think of it Simply

Imagine you need to explain to a robot how the world works. You cannot just show it everything — you need a language or structure to write down facts, rules, and relationships. That language/structure is Knowledge Representation. Just like you store notes in a notebook using words and diagrams, AI stores knowledge using logic, networks, and frames.

WHAT is KR?

A systematic way to structure and store information so that an AI system can:

  • Store facts about the world
  • Make logical inferences
  • Reason about situations
  • Answer queries
  • Solve problems intelligently

WHY do we need KR?

  • Machine Reasoning: Machines cannot reason without structured knowledge
  • Communication: Provides common vocabulary for AI systems
  • Inference: Enables deriving new facts from existing ones
  • Problem Solving: Foundation for expert systems and AI agents
  • Consistency: Ensures knowledge is organized and non-contradictory

WHEN to use KR?

  • Building expert systems (medical diagnosis, legal reasoning)
  • Natural language understanding
  • Robotics and planning
  • Semantic web and ontology design
  • Any AI system that needs to reason about the world

HOW is KR done?

  • Logic-based: Propositional Logic, Predicate Logic (FOPL)
  • Structured: Semantic Networks, Frames, Scripts
  • Procedural: Rules (IF-THEN), Production Systems
  • Hybrid: Combining multiple approaches

Properties of a Good Knowledge Representation

A good KR system must have:

  • Representational Adequacy: Ability to represent all kinds of knowledge needed in the domain
  • Inferential Adequacy: Ability to derive new knowledge from existing knowledge through inference
  • Inferential Efficiency: Ability to incorporate heuristics and control mechanisms to speed up inference
  • Acquisitional Efficiency: Ease of acquiring new knowledge (adding, modifying, deleting)

Q: Explain different types of Knowledge Representation techniques with examples. Compare their advantages and disadvantages.

Answer:

Knowledge Representation techniques are broadly classified into three categories:

1. Logic-Based Representations:

  • Propositional Logic: Uses simple true/false propositions connected by AND, OR, NOT, IMPLIES. Example: "It is raining" (P), "I carry umbrella" (Q), Rule: P → Q
  • Predicate Logic (FOPL): Extends propositional logic with predicates, variables, quantifiers. Example: ∀x (Student(x) → Studies(x)) meaning "All students study"

2. Structured Representations:

  • Semantic Networks: Graph with nodes (concepts) and labeled edges (relationships). Example: "Dog IS-A Animal", "Dog HAS-A Tail"
  • Frames: Data structures with slots (attributes) and fillers (values), support inheritance. Example: A "Car" frame with slots: Color, Make, Model, Engine
  • Scripts: Represent stereotypical event sequences. Example: Restaurant script with scenes: Entering, Ordering, Eating, Paying

3. Procedural Representations:

  • Production Rules: IF-THEN rules. Example: IF temperature > 100 THEN fever = true
  • Conceptual Dependency: Uses primitive acts to represent actions. Example: "John ate pizza" = INGEST(John, pizza, mouth)

2. Propositional Logic (Propositional Knowledge)

Propositional Logic (also called Boolean Logic or Sentential Logic) is the simplest form of logic where knowledge is represented as simple declarative statements (propositions) that are either TRUE or FALSE. These propositions are connected using logical connectives.

Simple Analogy
Think of Light Switches

Each proposition is like a light switch — it can only be ON (True) or OFF (False). You can combine switches: "Switch A AND Switch B must both be ON for the fan to work." That is exactly how propositional logic works — combining true/false statements with AND, OR, NOT.

Components of Propositional Logic

A. Propositions (Atomic Statements)

A proposition is a declarative sentence that is either true or false, but not both.

Valid Propositions
  • "Delhi is the capital of India" — TRUE (this is a valid proposition)
  • "2 + 3 = 6" — FALSE (this is a valid proposition, just false)
  • "It is raining today" — TRUE or FALSE depending on the day

NOT Propositions: "What time is it?" (question), "Close the door" (command), "x + 5 = 10" (has variable, truth depends on x)

B. Logical Connectives

Connective Symbol Name Meaning Example
NOT ¬ or ~ Negation Reverses truth value ¬P: "It is NOT raining"
AND ∧ or Λ Conjunction Both must be true P ∧ Q: "It is raining AND cold"
OR ∨ or V Disjunction At least one must be true P ∨ Q: "It is raining OR sunny"
IMPLIES → or ⇒ Implication If P then Q P → Q: "If raining THEN carry umbrella"
BICONDITIONAL ↔ or ⇔ Equivalence P if and only if Q P ↔ Q: "Pass IFF marks ≥ 40"

Important Equivalences (Laws)

Key Laws to Remember for Exams:

De Morgan's Laws: ¬(P ∧ Q) ≡ ¬P ∨ ¬Q   |   ¬(P ∨ Q) ≡ ¬P ∧ ¬Q

Distributive: P ∧ (Q ∨ R) ≡ (P ∧ Q) ∨ (P ∧ R)
Distributive: P ∨ (Q ∧ R) ≡ (P ∨ Q) ∧ (P ∨ R)

Limitations of Propositional Logic

Why Propositional Logic is NOT Enough

  • Cannot express relationships: Cannot say "John loves Mary" (needs predicates)
  • Cannot express generality: Cannot say "ALL students study" (needs quantifiers)
  • Cannot use variables: Each fact needs a separate proposition
  • Combinatorial explosion: For n objects with m properties, need n×m propositions
  • No functions: Cannot express "father of X"

This is why we need FOPL (First-Order Predicate Logic)!

Q: Explain Propositional Logic with examples. What are its limitations?

Answer:

Propositional Logic is a knowledge representation technique where knowledge is expressed as declarative statements (propositions) that are either TRUE or FALSE. Propositions are combined using logical connectives: NOT (¬), AND (∧), OR (∨), IMPLIES (→), and BICONDITIONAL (↔).

Example: Let P = "It is raining", Q = "I carry umbrella"

  • P ∧ Q: "It is raining AND I carry umbrella"
  • P → Q: "IF it is raining THEN I carry umbrella"

Limitations: Cannot express relationships between objects, cannot use variables or quantifiers (∀, ∃), cannot represent "All X have property Y", leads to combinatorial explosion for large domains. These limitations are overcome by Predicate Logic (FOPL).

3. First-Order Predicate Logic (FOPL)

First-Order Predicate Logic (FOPL), also called First-Order Logic (FOL), extends propositional logic by introducing predicates (to express properties and relationships), variables (to refer to objects generically), quantifiers (to express generality), and functions (to refer to specific objects indirectly). It is the most widely used KR technique in AI.

Simple Analogy
Propositional Logic vs FOPL — A Student's Perspective

Propositional Logic: Like writing "Rahul passed", "Priya passed", "Amit passed" — separate statement for each student.

FOPL: Like writing "All students who scored above 40 passed" — ONE statement covers ALL students! This is the power of FOPL.

Components of FOPL

ComponentDescriptionExamples
ConstantsSpecific objects in the domainJohn, Delhi, 5, India
VariablesGeneral objects (placeholders)x, y, z
PredicatesProperties or relationshipsStudent(x), Loves(x,y), Greater(x,y)
FunctionsMapping from objects to objectsfather(x), age(x), plus(x,y)
ConnectivesLogical operators∧, ∨, ¬, →, ↔
QuantifiersExpress scope over variables∀ (for all), ∃ (there exists)
EqualityTwo terms refer to same objectfather(John) = Robert

Quantifiers in Detail

Universal Quantifier (∀)

Meaning: "For ALL" or "For every"

Symbol: ∀x

Read as: "For all x" or "For every x"

Examples:

  • "All humans are mortal" → ∀x (Human(x) → Mortal(x))
  • "Every student studies" → ∀x (Student(x) → Studies(x))
  • "All birds can fly" → ∀x (Bird(x) → CanFly(x))

Key Rule: ∀ always uses IMPLICATION (→) to connect the condition and conclusion. Using ∧ would mean EVERYTHING in the universe is both the condition and conclusion!

Existential Quantifier (∃)

Meaning: "There exists" or "For some"

Symbol: ∃x

Read as: "There exists an x" or "For some x"

Examples:

  • "Some students are toppers" → ∃x (Student(x) ∧ Topper(x))
  • "There is a dog that is brown" → ∃x (Dog(x) ∧ Brown(x))
  • "Someone loves Mary" → ∃x Loves(x, Mary)

Key Rule: ∃ always uses AND (∧) to connect properties. Using → would be too weak — it would be satisfied even if nothing has the property!

Common Exam Mistake: Students often confuse when to use → vs ∧ with quantifiers. Remember: ∀ uses → (implication) and ∃ uses ∧ (conjunction). Getting this wrong will cost marks!

Converting English to FOPL — Step-by-Step Examples

Conversion Walkthrough
Example 1: "All dogs are animals"

Step 1: Identify the quantifier → "All" means ∀

Step 2: Identify subject → dogs → Dog(x)

Step 3: Identify property → animals → Animal(x)

Step 4: Use → with ∀

Result: ∀x (Dog(x) → Animal(x))

Example 2: "Some students like Mathematics"

Step 1: "Some" means ∃

Step 2: Subject: Student(x)

Step 3: Property: Likes(x, Mathematics)

Step 4: Use ∧ with ∃

Result: ∃x (Student(x) ∧ Likes(x, Mathematics))

Example 3: "Every child loves its mother"

Step 1: "Every" means ∀

Step 2: Subject: Child(x)

Step 3: Property: Loves(x, mother(x)) — using function mother(x)

Step 4: Use → with ∀

Result: ∀x (Child(x) → Loves(x, mother(x)))

Example 4: "Not all birds can fly"

Step 1: "Not all" = ¬∀ = ∃¬ (negation of universal)

Step 2: Subject: Bird(x), Property: CanFly(x)

Result: ¬∀x (Bird(x) → CanFly(x)) OR equivalently ∃x (Bird(x) ∧ ¬CanFly(x))

"There exists a bird that cannot fly" (e.g., Penguin)

Example 5: "Everyone who is a friend of John is a friend of Mary"

Step 1: "Everyone" means ∀

Step 2: Condition: Friend(x, John)

Step 3: Conclusion: Friend(x, Mary)

Result: ∀x (Friend(x, John) → Friend(x, Mary))

Step 1: "There is a student" = ∃x Student(x)

Step 2: "in every subject" = ∀y Subject(y)

Step 3: Property: ScoresHighest(x, y)

Result: ∃x (Student(x) ∧ ∀y (Subject(y) → ScoresHighest(x, y)))

Negation with Quantifiers

Important Negation Rules:

¬∀x P(x) ≡ ∃x ¬P(x)    ("Not all have P" = "Some don't have P")
¬∃x P(x) ≡ ∀x ¬P(x)    ("None has P" = "All don't have P")

4. Well-Formed Formulas (WFF)

A Well-Formed Formula (WFF) is a syntactically correct expression in predicate logic that follows specific formation rules. Not every combination of symbols is a valid formula — only those that follow the grammar rules of the logic language are WFFs.

Think of it Simply

Just like "I am student a" is not a valid English sentence (wrong grammar), similarly "∀ ∧ P(x Q" is not a valid logical formula. A WFF is like a grammatically correct sentence in the language of logic.

Rules for Constructing WFFs

WFF Formation Rules
  1. Atomic Formula: A predicate with the correct number of terms is a WFF. Example: P(x), Loves(John, Mary), Greater(x, 5)
  2. Negation: If α is a WFF, then ¬α is a WFF. Example: ¬P(x)
  3. Conjunction: If α and β are WFFs, then (α ∧ β) is a WFF. Example: (P(x) ∧ Q(y))
  4. Disjunction: If α and β are WFFs, then (α ∨ β) is a WFF. Example: (P(x) ∨ Q(y))
  5. Implication: If α and β are WFFs, then (α → β) is a WFF
  6. Biconditional: If α and β are WFFs, then (α ↔ β) is a WFF
  7. Universal Quantification: If α is a WFF, then ∀x α is a WFF
  8. Existential Quantification: If α is a WFF, then ∃x α is a WFF
  9. Nothing else is a WFF (closed world assumption for syntax)

Examples of Valid and Invalid WFFs

ExpressionValid WFF?Reason
P(x)YesAtomic formula — predicate with argument
∀x (Human(x) → Mortal(x))YesProper quantification with valid sub-WFF
∃x (Dog(x) ∧ Barks(x))YesExistential quantification with conjunction
¬∀x P(x)YesNegation of a valid WFF
∀x ∨ P(x)No∨ cannot come directly after quantifier without operands
P(x) Q(y)NoMissing connective between two formulas

Free and Bound Variables

Understanding Variable Binding

Bound Variable: A variable that is within the scope of a quantifier. Example: In ∀x P(x), x is bound by ∀.

Free Variable: A variable not bound by any quantifier. Example: In P(x) ∧ ∀y Q(y), x is free and y is bound.

A closed WFF (sentence): A WFF with NO free variables. Only closed WFFs have a definite truth value.

Exam Question
4 Marks

Q: What is a Well-Formed Formula (WFF)? Give the rules for constructing WFFs in FOPL with examples.

Answer:

A Well-Formed Formula (WFF) is a syntactically valid expression in predicate logic formed by following strict formation rules. It ensures that logical expressions are meaningful and can be evaluated.

Rules:

  1. An atomic formula (predicate with correct terms) is a WFF. E.g., Likes(John, Pizza)
  2. If α is WFF, then ¬α is WFF. E.g., ¬Likes(John, Pizza)
  3. If α, β are WFFs, then α ∧ β, α ∨ β, α → β, α ↔ β are WFFs
  4. If α is WFF and x is a variable, then ∀x α and ∃x α are WFFs
  5. Nothing else is a WFF

Examples: ∀x (Student(x) → Learns(x)) is a valid WFF. But ∀ ∧ P(x) is NOT a WFF because the syntax is wrong.

5. Inference Rules & Theorem Proving

Inference is the process of deriving new logical conclusions (new knowledge) from existing knowledge using established rules. Theorem Proving is the systematic application of inference rules to prove that a given statement (theorem) logically follows from a set of known facts (axioms).

Think of it Simply

If you know "All B.Tech students study AI" and "Rahul is a B.Tech student", you can infer "Rahul studies AI". You just applied an inference rule (Modus Ponens) without even realizing it! Theorem proving is formalizing this process so computers can do it too.

Key Inference Rules

1. Modus Ponens (The Most Important Rule!)

Modus Ponens (MP) — "Mode of Affirming"

If P → Q is true, and P is true, then Q is true
Notation: P, P → Q ⊢ Q
Example STUDENT

Premise 1: If it rains, the ground is wet. (Rain → WetGround)

Premise 2: It is raining. (Rain)

Conclusion: The ground is wet. (WetGround) ✓

2. Modus Tollens

Modus Tollens (MT) — "Mode of Denying"

If P → Q is true, and ¬Q is true, then ¬P is true
Notation: P → Q, ¬Q ⊢ ¬P
Example STUDENT

Premise 1: If it rains, the ground is wet. (Rain → WetGround)

Premise 2: The ground is NOT wet. (¬WetGround)

Conclusion: It is NOT raining. (¬Rain) ✓

3. And-Elimination (Simplification)

If P ∧ Q is true, then P is true (and Q is true)
P ∧ Q ⊢ P   (also P ∧ Q ⊢ Q)

4. And-Introduction (Conjunction)

If P is true and Q is true, then P ∧ Q is true
P, Q ⊢ P ∧ Q

5. Or-Introduction (Addition)

If P is true, then P ∨ Q is true (for any Q)
P ⊢ P ∨ Q

6. Universal Instantiation (UI)

If ∀x P(x) is true, then P(c) is true for any constant c
∀x P(x) ⊢ P(a) (substitute any specific object)
Example

Given: ∀x (Human(x) → Mortal(x)) — "All humans are mortal"

Apply UI with x = Socrates: Human(Socrates) → Mortal(Socrates)

7. Existential Instantiation (EI)

If ∃x P(x) is true, then P(c) is true for some new constant c
∃x P(x) ⊢ P(c) (c must be a NEW constant not used before)

8. Hypothetical Syllogism (Chain Rule)

If P → Q and Q → R, then P → R
P → Q, Q → R ⊢ P → R

Summary of All Inference Rules

RuleGivenConcludeMnemonic
Modus PonensP, P → QQ"If P then Q; P is true; so Q"
Modus TollensP → Q, ¬Q¬P"If P then Q; Q is false; so not P"
And-EliminationP ∧ QP (or Q)"Both true, so each is true"
And-IntroductionP, QP ∧ Q"Each true, so both true"

Forward Chaining vs Backward Chaining

Forward Chaining

Strategy: Start from known facts, apply rules to derive new facts until goal is reached

Direction: Data-driven (bottom-up)

Process: Facts → Rules → New Facts → ... → Goal

Example:

Facts: Human(Socrates), ∀x(Human(x)→Mortal(x))

Apply MP → Mortal(Socrates) ✓

Used in: Production systems, CLIPS, business rule engines

Backward Chaining

Strategy: Start from the goal, find rules that can prove it, check if their premises are known

Direction: Goal-driven (top-down)

Process: Goal → What rules lead here? → Are premises true?

Example:

Goal: Mortal(Socrates)?

Find rule: Human(x)→Mortal(x). Need: Human(Socrates)? Yes! ✓

Used in: Prolog, diagnostic systems, Prolog-based expert systems

Theorem Proving — Worked Example

Prove: Socrates is Mortal
Step 1: State the Knowledge Base

KB1: ∀x (Human(x) → Mortal(x)) — "All humans are mortal"

KB2: Human(Socrates) — "Socrates is a human"

Goal: Prove Mortal(Socrates)

Step 2: Apply Universal Instantiation on KB1

Substitute x = Socrates in KB1:

Human(Socrates) → Mortal(Socrates) ... (KB3)

Step 3: Apply Modus Ponens using KB2 and KB3

From KB2: Human(Socrates) is TRUE

From KB3: Human(Socrates) → Mortal(Socrates)

By Modus Ponens: Mortal(Socrates) is TRUE ✓

Theorem Proved! Socrates is mortal. QED.

Exam Question
8 Marks

Q: Explain the following inference rules with examples: (i) Modus Ponens (ii) Modus Tollens (iii) Universal Instantiation. Also prove that "Socrates is mortal" given "All humans are mortal" and "Socrates is a human".

Answer:

(i) Modus Ponens: If P → Q is true and P is true, then Q must be true. Example: "If it rains, ground is wet" + "It rains" ⇒ "Ground is wet"

(ii) Modus Tollens: If P → Q is true and ¬Q is true, then ¬P must be true. Example: "If it rains, ground is wet" + "Ground is NOT wet" ⇒ "It is NOT raining"

(iii) Universal Instantiation: If ∀x P(x) is true, then P(c) is true for any constant c. Example: "All dogs bark" ⇒ "Rex barks" (where Rex is a specific dog)

Proof:

  1. Given: ∀x (Human(x) → Mortal(x)) and Human(Socrates)
  2. Apply UI (x = Socrates): Human(Socrates) → Mortal(Socrates)
  3. Apply Modus Ponens: Human(Socrates) is true + Human(Socrates) → Mortal(Socrates)
  4. Therefore: Mortal(Socrates) is TRUE. Proved.

6. Resolution

Resolution is a powerful inference rule used for automated theorem proving. It works by refutation — to prove a statement, we negate it and add it to the knowledge base, then try to derive a contradiction (empty clause). If we reach a contradiction, the original statement must be true.

Simple Analogy
Proof by Contradiction — Like a Court Case

In a court, to prove someone is innocent, the lawyer sometimes assumes they are guilty and then shows this leads to impossible contradictions. When the assumption leads to absurdity, the opposite (innocence) must be true. Resolution works the same way!

Steps for Resolution (Very Important for Exams!)

Resolution Procedure
  1. Convert to Clause Form: Convert all statements to Conjunctive Normal Form (CNF) — conjunction of disjunctions of literals
  2. Negate the Goal: Add the negation of what you want to prove
  3. Apply Resolution Rule: Find two clauses with complementary literals (P and ¬P), resolve them to create a new clause
  4. Repeat: Keep resolving until you derive the empty clause (contradiction) or no new clauses can be generated
  5. Conclusion: If empty clause is derived → original goal is PROVED. If not → goal cannot be proved.

Conversion to Clause Form (CNF)

Steps to Convert to CNF
Step 1: Eliminate Biconditionals (↔)

P ↔ Q becomes (P → Q) ∧ (Q → P)

Step 2: Eliminate Implications (→)

P → Q becomes ¬P ∨ Q

Step 3: Move NOT inward (using De Morgan's)

¬(P ∧ Q) becomes ¬P ∨ ¬Q

¬(P ∨ Q) becomes ¬P ∧ ¬Q

¬∀x P(x) becomes ∃x ¬P(x)

¬∃x P(x) becomes ∀x ¬P(x)

Step 4: Standardize Variables

Rename variables so each quantifier uses a unique variable name

Step 5: Skolemization (Remove ∃)

Replace existentially quantified variables with Skolem constants/functions

∃x P(x) becomes P(a) where 'a' is a new Skolem constant

∀x ∃y P(x,y) becomes ∀x P(x, f(x)) where f is a Skolem function

Step 6: Drop Universal Quantifiers

Since all remaining variables are universally quantified, we can drop ∀

Step 7: Distribute ∨ over ∧

P ∨ (Q ∧ R) becomes (P ∨ Q) ∧ (P ∨ R)

Step 8: Split into Clauses

Each conjunct becomes a separate clause. Final form: set of clauses where each clause is a disjunction of literals.

The Resolution Rule

Resolution Rule:

Clause 1: A ∨ P

What is Unification?

Unification is the process of finding a substitution that makes two expressions identical. It is essential for resolution in FOPL.

Examples:

  • Unify P(x) with P(John): substitution {x/John} ✓

7. Semantic Networks

A Semantic Network is a knowledge representation technique that uses a graph structure consisting of nodes (representing concepts/objects) connected by labeled edges (representing relationships between concepts). It provides a visual and intuitive way to represent knowledge.

Simple Analogy
Think of a Mind Map

When you create a mind map for studying, you put the main topic in the center and draw lines to related subtopics with labels describing how they are connected. A Semantic Network is exactly that — but formalized for computers! Each bubble is a node, each arrow is a labeled relationship.

Inheritance in Semantic Networks

Property Inheritance

One of the most powerful features of semantic networks is inheritance — properties defined at a higher level automatically apply to lower levels.

Example:

  • Animal HAS property "Breathes" → Since Dog IS-A Mammal IS-A Animal, Dog also "Breathes" (inherited!)
  • Bird CAN "Fly" → Sparrow IS-A Bird, so Sparrow CAN "Fly" (inherited)
  • BUT Penguin IS-A Bird, yet Penguin CANNOT "Fly" → This is an exception that overrides the inherited property

Rule: Properties are inherited from parent to child, but local (specific) properties override inherited (general) ones.

Slots in Semantic Networks

What are Slots?

Slots are the property-value pairs associated with nodes. Each relationship from a node to a property or another node fills a "slot".

  • Dog has slots: legs=4, sound="bark", covering="fur"
  • Bird has slots: legs=2, covering="feathers", canFly=true
  • Slots can hold: values, pointers to other nodes, or default values

Advantages

  • Visual and intuitive representation
  • Easy to understand relationships
  • Supports inheritance (reduces redundancy)
  • Natural way to represent taxonomies
  • Easy to add new knowledge

Disadvantages

  • No standard formal semantics
  • Cannot handle exceptions well
  • Limited inference capabilities
  • Becomes complex for large knowledge bases
  • Cannot represent disjunction, negation easily

8. Frames

A Frame (proposed by Marvin Minsky in 1975) is a data structure for representing a stereotypical situation or object. It consists of a collection of slots (attributes) and fillers (values), along with attached procedures called demons (if-needed, if-added). Frames support inheritance through IS-A hierarchies.

Simple Analogy
Think of a Form/Template

When you fill a college admission form, it has predefined fields: Name, Roll No., Branch, Year, etc. Some fields have default values (Year: 1st). A Frame is exactly like a form — it has predefined slots that can be filled with specific values. And just like all B.Tech forms share the same template, Frames support inheritance (child frames inherit slots from parent frames).

9. Conceptual Dependency (CD) Theory

Conceptual Dependency (CD) theory, proposed by Roger Schank (1972), represents the meaning of natural language sentences using a small set of primitive acts. The core idea is that sentences with the same meaning should have the same CD representation, regardless of the words used. This provides a language-independent meaning representation.

Simple Analogy
Think of Emoji Communication

Just like emojis express ideas regardless of language (a heart emoji means love in English, Hindi, or Japanese), CD uses universal "primitive acts" to express meaning regardless of which words are used. "John ate pizza", "John consumed pizza", "John had pizza" — all become the SAME CD representation!

Primitive Acts in CD

Schank defined 11 primitive acts that can represent any physical or mental action:

Primitive ActMeaningExample
ATRANSTransfer of abstract relationship (give, take, buy)John gave the book to Mary = ATRANS(book, John, Mary)
PTRANSTransfer of physical location (go, move, walk)John went to school = PTRANS(John, home, school)
MTRANSTransfer of mental information (tell, learn, read)John told Mary a story = MTRANS(story, John, Mary)
MBUILDBuild new information (think, decide, imagine)John decided to go = MBUILD(decision, John)
PROPELApply force to object (push, pull, throw)John pushed the cart = PROPEL(John, cart, force)
MOVEMove a body partJohn raised his hand = MOVE(John, hand, up)
INGESTTake something inside body (eat, drink, breathe)John ate pizza = INGEST(John, pizza, mouth)
EXPELExpel from body (cry, spit, exhale)John cried = EXPEL(John, tears, eyes)
GRASPGrasp an object (hold, grab, release)John grabbed the ball = GRASP(John, ball)
ATTENDFocus sense organ (look, listen, smell)John looked at the painting = ATTEND(John, eyes, painting)
SPEAKProduce soundJohn said "hello" = SPEAK(John, "hello")

10. Scripts

A Script (proposed by Schank & Abelson, 1977) is a structured knowledge representation that describes a stereotypical sequence of events in a particular context. It captures commonly occurring situations with typical roles, entry conditions, scenes (ordered events), and results.

Simple Analogy
Think of a Movie Script

Just like a movie script describes who the characters are, what scenes happen in what order, and what the outcome is — a Script in AI describes a standard situation. You know what happens at a restaurant: enter, sit, order, eat, pay, leave. This "common knowledge" is captured in a Script.

2-Mark Questions (Short Answer)

2 Marks

Q1: Define Knowledge Representation.

A: Knowledge Representation (KR) is the study of how to encode knowledge about the world in a form that a computer system can use for automated reasoning and problem solving. It acts as a bridge between real-world knowledge and machine understanding.

2 Marks

Q2: What is the difference between ∀ and ∃ quantifiers?

A: ∀ (Universal Quantifier) means "for all" — the statement holds for every object in the domain. ∃ (Existential Quantifier) means "there exists" — the statement holds for at least one object. Example: ∀x Dog(x)→Animal(x) vs ∃x Dog(x)∧White(x).

2 Marks

Q3: What is Modus Ponens?

A: Modus Ponens is an inference rule: If P → Q is true and P is true, then Q must be true. Example: "If it rains, ground is wet" + "It rains" = "Ground is wet".

2 Marks

Q4: What is a Well-Formed Formula?

A: A WFF is a syntactically correct expression in predicate logic that follows specific formation rules. It ensures logical expressions are meaningful. Example: ∀x(P(x)→Q(x)) is a WFF, but ∀∧P(x) is not.

2 Marks

Q5: What is unification in resolution?

A: Unification is the process of finding a substitution that makes two expressions identical. It is used in resolution to match complementary literals. Example: Unifying P(x) with P(John) gives substitution {x/John}.

2 Marks

Q6: Name the 4 properties of a good KR system.

A: (1) Representational Adequacy (2) Inferential Adequacy (3) Inferential Efficiency (4) Acquisitional Efficiency.

2 Marks

Q7: What is the IS-A relationship in Semantic Networks?

A: IS-A represents a class hierarchy or subclass relationship between nodes. "Dog IS-A Animal" means Dog is a subclass of Animal, inheriting all properties of Animal.

2 Marks

Q8: What are demons in Frames?

A: Demons are attached procedures in frames that are triggered automatically by events. IF-NEEDED demons execute when a slot value is required but missing. IF-ADDED demons execute when a new value is added to a slot.

4-Mark Questions

4 Marks

Q9: Explain the limitations of propositional logic and how FOPL overcomes them.

A: Propositional logic limitations: (1) Cannot express relationships between objects (2) No variables — each fact needs separate proposition (3) No quantifiers — cannot say "all" or "some" (4) No functions. FOPL overcomes these by introducing: predicates (Loves(x,y) for relationships), variables (x,y,z for generalization), quantifiers (∀ for all, ∃ for some), and functions (father(x)). Example: Instead of separate propositions for each student, FOPL uses ∀x(Student(x)→Studies(x)).

Q12: What is forward chaining? How is it different from backward chaining?

A: Forward Chaining is a data-driven reasoning strategy that starts from known facts and applies inference rules to derive new facts until the goal is reached. It works bottom-up.

Backward Chaining is goal-driven: starts from the goal and works backward to find supporting facts. It works top-down.

Example: Given Human(Socrates) and ∀x(Human(x)→Mortal(x)), Forward chaining derives Mortal(Socrates) by matching facts to rules. Backward chaining starts with "Is Mortal(Socrates)?" and looks for rules that conclude it.

Forward chaining is used in production systems (CLIPS), backward chaining in Prolog and diagnostic systems.

(2) PTRANS — Transfer of physical location. "John went to school" = PTRANS(John, home, school)

(3) MTRANS — Transfer of mental information. "Teacher taught students" = MTRANS(knowledge, teacher, students)

(4) INGEST — Taking something inside the body. "Ram ate rice" = INGEST(Ram, rice, mouth)

4 Marks

Q14: What are the steps to convert a FOPL statement to clause form for resolution?

A: Steps: (1) Eliminate biconditionals: P↔Q becomes (P→Q)∧(Q→P) (2) Eliminate implications: P→Q becomes ¬P∨Q (3) Move NOT inward using De Morgan's laws (4) Standardize variables (unique names) (5) Skolemize — replace ∃ with Skolem constants/functions (6) Drop universal quantifiers (7) Distribute ∨ over ∧ (8) Split into clauses (each conjunct is a clause).

Q17: Using resolution refutation, prove: "John likes pizza" given: (1) Everyone who is hungry eats food (2) Everyone who eats food likes food (3) Pizza is food (4) John is hungry.

A:

Step 1 — FOPL:

F1: ∀x (Hungry(x) → Eats(x, Food))

F2: ∀x ∀y (Eats(x,y) ∧ Food(y) → Likes(x,y))

F3: Food(Pizza)

F4: Hungry(John)

Step 2 — Convert to Clause Form:

C1: ¬Hungry(x) ∨ Eats(x, Food) [from F1]

C2: ¬Eats(x,y) ∨ ¬Food(y) ∨ Likes(x,y) [from F2]

C3: Food(Pizza) [from F3]

C4: Hungry(John) [from F4]

Step 3 — Negate Goal:

C5: ¬Likes(John, Pizza)

Step 4 — Resolution:

Resolve C2 and C5 {x/John, y/Pizza}: ¬Eats(John, Pizza) ∨ ¬Food(Pizza) → C6

Resolve C6 and C3: ¬Eats(John, Pizza) → C7

Resolve C1 and C4 {x/John}: Eats(John, Food) → C8

With Food=Pizza (from context): Eats(John, Pizza) → C8'

Resolve C7 and C8': { } Empty Clause!

Contradiction! Therefore Likes(John, Pizza) is PROVED.

Q21: Explain Skolemization with an example.

A: Skolemization is the process of removing existential quantifiers by replacing existentially quantified variables with Skolem constants or Skolem functions.

Case 1 — No preceding universal quantifier: ∃x P(x) becomes P(a) where 'a' is a new Skolem constant.

Case 2 — Under a universal quantifier: ∀x ∃y Loves(x,y) becomes ∀x Loves(x, f(x)) where f is a Skolem function (the loved one depends on x).

Example: "Everyone has a mother" — ∀x ∃y Mother(y,x) becomes ∀x Mother(f(x), x) where f(x) is the mother function.

F → T is True because implication represents a promise: "If P then Q". When P is false (the condition doesn't occur), the promise is NOT broken regardless of Q's value. Q24: Explain all inference rules used in theorem proving. Demonstrate forward chaining and backward chaining with an example.

A: Inference Rules:

(1) Modus Ponens: P, P→Q |- Q (2) Modus Tollens: P→Q, ¬Q |- ¬P (3) And-Elimination: P∧Q |- P (4) And-Introduction: P, Q |- P∧Q (5) Or-Introduction: P |- P∨Q (6) Hypothetical Syllogism: P→Q, Q→R |- P→R (7) Universal Instantiation: ∀x P(x) |- P(c) (8) Existential Instantiation: ∃x P(x) |- P(c)

Knowledge Base:

R1: ∀x(Bird(x) ∧ ¬Abnormal(x) → CanFly(x))

R2: ∀x(CanFly(x) → HasWings(x))

F1: Bird(Tweety), F2: ¬Abnormal(Tweety)

Forward Chaining: Start from facts, apply rules forward.

Step 1: Match R1 with F1 & F2 (x=Tweety) → derive CanFly(Tweety)

Step 2: Match R2 with CanFly(Tweety) → derive HasWings(Tweety)

Backward Chaining: Start from goal, work backward.

Goal: HasWings(Tweety)?

Step 1: Find rule concluding HasWings → R2 needs CanFly(Tweety)

Step 2: Find rule concluding CanFly → R1 needs Bird(Tweety) ∧ ¬Abnormal(Tweety)

Step 3: Check F1: Bird(Tweety)? YES. Check F2: ¬Abnormal(Tweety)? YES.

Goal proved! HasWings(Tweety) = TRUE.

Unit 2 — Key Takeaways for Exam

  • KR is how we encode knowledge for AI systems. 4 properties: Representational, Inferential, Efficiency, Acquisitional.
  • Propositional Logic: Simple T/F statements with connectives. Limited — no variables, no quantifiers.
  • FOPL: Predicates + Variables + Quantifiers (∀ uses →, ∃ uses ∧). Most powerful logic-based KR.
  • WFF: Syntactically correct formula in logic. Follow formation rules strictly.
  • Inference Rules: Modus Ponens is king! Also know Modus Tollens, UI, EI.
  • Resolution: Negate goal, convert to CNF, resolve until empty clause = proved. Know the 8-step CNF conversion.
  • Semantic Networks: Nodes + labeled edges. IS-A for hierarchy, HAS-A for properties. Inheritance!
  • Frames: Slots + Fillers + Defaults + Demons. Proposed by Minsky. Supports inheritance.
  • Scripts: Stereotypical event sequences. Entry conditions, Roles, Props, Scenes, Results. Classic example: Restaurant.
  • CD: 11 primitive acts (ATRANS, PTRANS, MTRANS, INGEST...). Language-independent meaning.
