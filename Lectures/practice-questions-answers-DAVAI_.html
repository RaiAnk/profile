Question 1
4 Marks CO3 L2

Define Range and Quartiles. For the dataset: [8, 12, 15, 19, 23, 27, 31, 35], calculate Q1, Q2 (Median), and Q3.

Answer:

Definitions:

  • Range: The difference between the maximum and minimum values in a dataset. Range = Max - Min
  • Quartiles: Values that divide a sorted dataset into four equal parts.
    • Q1 (First Quartile): 25th percentile - separates lowest 25% of data
    • Q2 (Second Quartile/Median): 50th percentile - middle value
    • Q3 (Third Quartile): 75th percentile - separates lowest 75% of data

Calculation:

Dataset (already sorted): [8, 12, 15, 19, 23, 27, 31, 35], n = 8

Range: 35 - 8 = 27

Q2 (Median):

For even n: Median = (4th + 5th values) / 2 = (19 + 23) / 2 = 21

Q1: Median of lower half [8, 12, 15, 19]

Q1 = (12 + 15) / 2 = 13.5

Q3: Median of upper half [23, 27, 31, 35]

Q3 = (27 + 31) / 2 = 29

For the following employee ages dataset: [28, 32, 35, 38, 40, 42, 45, 48, 52, 58]

  1. Calculate the Mean and Standard Deviation.
  2. Determine which data points lie outside 1.5 standard deviations from the mean.
  3. Interpret what this analysis reveals about the age distribution.

Answer:

(i) Mean and Standard Deviation:

Dataset: [28, 32, 35, 38, 40, 42, 45, 48, 52, 58], n = 10

Mean (x-bar): Sum / n = (28+32+35+38+40+42+45+48+52+58) / 10 = 418 / 10 = 41.8

Standard Deviation:

Step 1: Calculate squared deviations from mean:

(28-41.8)^2 + (32-41.8)^2 + ... + (58-41.8)^2 = 190.44 + 96.04 + 46.24 + 14.44 + 3.24 + 0.04 + 10.24 + 38.44 + 104.04 + 262.44 = 765.6

Step 2: Variance = 765.6 / 10 = 76.56

Step 3: SD = sqrt(76.56) = 8.75

(ii) Data points outside 1.5 SD:

Lower bound = Mean - 1.5*SD = 41.8 - 1.5(8.75) = 41.8 - 13.125 = 28.675

Upper bound = Mean + 1.5*SD = 41.8 + 1.5(8.75) = 41.8 + 13.125 = 54.925

Data points outside [28.675, 54.925]:

28 (below lower bound) and 58 (above upper bound)

(iii) Interpretation:

  • The mean age of 41.8 years indicates a middle-aged workforce
  • SD of 8.75 shows moderate spread in ages
  • Only 2 out of 10 values (20%) lie outside 1.5 SD, indicating most employees are clustered around the mean
  • The distribution appears relatively symmetric with slight outliers at both extremes

8 Marks CO3 L4

A factory produces bolts with diameters following a Normal Distribution with mean = 10mm and standard deviation = 0.5mm. Quality control accepts bolts between 9.2mm and 10.8mm.

  1. What percentage of bolts will be rejected?
  2. If 10,000 bolts are produced daily, how many are expected to be rejected?
  3. What diameter range contains the middle 95% of bolts?

Answer:

(i) Percentage of bolts rejected:

Given: Mean = 10mm, SD = 0.5mm

Calculate Z-scores for acceptance limits:

Z(9.2) = (9.2 - 10) / 0.5 = -1.6

Z(10.8) = (10.8 - 10) / 0.5 = +1.6

From Z-table: P(-1.6 < Z < 1.6) = 0.9452 (approximately 94.52%)

Rejection rate: 100% - 94.52% = 5.48%

(ii) Expected rejected bolts from 10,000:

Rejected = 10,000 x 0.0548 = 548 bolts

(iii) Range for middle 95%:

For 95% confidence, Z = +/- 1.96

Lower limit = 10 - (1.96 x 0.5) = 10 - 0.98 = 9.02mm

Upper limit = 10 + (1.96 x 0.5) = 10 + 0.98 = 10.98mm

Middle 95% of bolts have diameter: 9.02mm to 10.98mm

Question 5
4 Marks CO3 L2

Explain the difference between Population Standard Deviation and Sample Standard Deviation. When should each be used? Write their formulas.

Answer:

Aspect Population SD Sample SD
Symbol sigma s
Denominator N (population size) n-1 (Bessel's correction)
When to use When data includes entire population When data is a sample from larger population

Formulas:

Population SD: sigma = sqrt[ sum(xi - mu)^2 / N ]

Sample SD: s = sqrt[ sum(xi - x-bar)^2 / (n-1) ]

Why n-1? Bessel's correction compensates for the bias when estimating population variance from a sample. Using n would underestimate the true population variance.

  1. Calculate the correlation coefficient between marks and attendance.

import numpy as np

# Data
marks = np.array([78, 85, 62, 91, 73, 88, 95, 67])
attendance = np.array([85, 90, 70, 95, 80, 88, 92, 75])

# (i) Correlation coefficient
correlation_matrix = np.corrcoef(marks, attendance)
correlation = correlation_matrix[0, 1]
print(f"Correlation coefficient: {correlation:.4f}")

# (ii) Students with marks above median
median_marks = np.median(marks)
print(f"\nMedian marks: {median_marks}")
above_median = marks[marks > median_marks]
print(f"Marks above median: {above_median}")
# With indices
indices = np.where(marks > median_marks)[0]
print(f"Student indices (0-based): {indices}")

# (iii) Percentage with attendance >= 80%
high_attendance = np.sum(attendance >= 80)
percentage = (high_attendance / len(attendance)) * 100
print(f"\nStudents with attendance >= 80%: {high_attendance}")
print(f"Percentage: {percentage:.2f}%")

Explain the 68-95-99.7 Rule (Empirical Rule) in Normal Distribution. The heights of adult males in a city follow a normal distribution with mean = 170 cm and standard deviation = 6 cm.

  1. What percentage of males have heights between 158 cm and 182 cm?
  2. What percentage of males are taller than 176 cm?
  3. If the city has 50,000 adult males, how many are expected to have heights below 158 cm?
  4. What height value separates the tallest 2.5% of males from the rest?

Answer:

68-95-99.7 Rule (Empirical Rule):

For a normal distribution:

  • 68% of data falls within 1 standard deviation of mean (mean +/- 1 SD)
  • 95% of data falls within 2 standard deviations of mean (mean +/- 2 SD)
  • 99.7% of data falls within 3 standard deviations of mean (mean +/- 3 SD)

Given: Mean = 170 cm, SD = 6 cm

(i) Percentage between 158 cm and 182 cm:

158 cm = 170 - 12 = 170 - 2(6) = Mean - 2 SD

182 cm = 170 + 12 = 170 + 2(6) = Mean + 2 SD

By empirical rule, 95% of males have heights between 158 cm and 182 cm.

(ii) Percentage taller than 176 cm:

176 cm = 170 + 6 = Mean + 1 SD

68% are within 1 SD, so 32% are outside

Since normal distribution is symmetric: 32% / 2 = 16% on each tail

Percentage taller than 176 cm = 16%

(iii) Expected males with height below 158 cm:

158 cm = Mean - 2 SD

95% within 2 SD means 5% outside, with 2.5% below lower limit

Expected count = 50,000 x 0.025 = 1,250 males

(iv) Height separating top 2.5%:

Top 2.5% corresponds to Z = +2 (since 97.5% below this point)

Height = Mean + 2*SD = 170 + 2(6) = 170 + 12

Height cutoff = 182 cm

  1. A scatter plot showing relationship between study_hours and scores

import matplotlib.pyplot as plt

study_hours = [2, 3, 4, 5, 6, 7, 8]

Answer:

Tableau Architecture:

1. Data Layer (Data Connectors)

  • Connects to various data sources (Excel, SQL, Cloud databases)
  • Supports live connections and data extracts
  • Handles data preparation and cleaning

2. Analytics Layer (VizQL Engine)

  • Proprietary Visual Query Language converts actions to queries
  • Performs calculations, aggregations, and transformations
  • Renders visualizations in real-time

3. Sharing Layer (Tableau Server/Cloud)

  • Publishes and shares dashboards
  • Manages user access and permissions
  • Enables collaboration and scheduling

  2. Create Worksheets:

  5. Add Filters: Drag Department field, select "Show Filter"

Question 17
8 Marks CO5 L3

Build a Simple Linear Regression model for employee productivity prediction.

Experience (years): [1, 2, 3, 4, 5, 6, 7, 8]
Productivity Score: [55, 60, 68, 72, 78, 82, 85, 88]

Answer:

import numpy as np
from sklearn.linear_model import LinearRegression
import matplotlib.pyplot as plt

# Data
X = np.array([1, 2, 3, 4, 5, 6, 7, 8]).reshape(-1, 1)  # Features (2D array)
y = np.array([55, 60, 68, 72, 78, 82, 85, 88])         # Target

# (i) Build Linear Regression Model
model = LinearRegression()
model.fit(X, y)

# (ii) Get Regression Equation
slope = model.coef_[0]
intercept = model.intercept_
print(f"Regression Equation: y = {slope:.2f}x + {intercept:.2f}")

# (iii) Predict for 10 years experience
prediction_10 = model.predict([[10]])
print(f"Predicted productivity for 10 years: {prediction_10[0]:.2f}")

# (iv) Calculate R-squared
r_squared = model.score(X, y)
print(f"R-squared Score: {r_squared:.4f}")

# Plot
plt.figure(figsize=(8, 6))
plt.scatter(X, y, color='blue', s=100, label='Actual Data')
plt.plot(X, model.predict(X), color='red', linewidth=2, label='Regression Line')
plt.scatter([10], prediction_10, color='green', s=150, marker='*', label='Prediction (10 yrs)')
plt.xlabel('Experience (years)')
plt.ylabel('Productivity Score')
plt.title('Employee Productivity vs Experience')
plt.legend()
plt.grid(True, alpha=0.3)
plt.show()

Output:

Predicted productivity for 10 years: 99.79
R-squared Score: 0.9911

Interpretation:

  • R-squared (0.9911): 99.11% of variance in productivity is explained by experience - excellent fit
Question 18
8 Marks CO5 L4

Explain Training Error, Test Error, and Cross-Validation concepts.

Answer:

(i) Training and Test Error vs Model Complexity:

    Error
      |
      |  \                           _____ Test Error
      |   \                    ____/
      |    \              ____/
      |     \____    ____/
      |          \__/  <-- Sweet Spot (Optimal Complexity)
      |           ____________________ Training Error
      |
      +---------------------------------> Model Complexity
           Simple                   Complex
  • Training Error: Decreases as complexity increases (model memorizes data)
  • Test Error: Initially decreases, then increases (overfitting)
  • Sweet Spot: Where test error is minimum - optimal model complexity

(ii) Identifying Overfitting and Underfitting:

Condition Training Error Test Error Issue
UnderfittingHighHighHigh Bias
Good FitLowLowBalanced
OverfittingVery LowHighHigh Variance

(iii) Cross-Validation and K-Fold:

Cross-Validation: Technique to evaluate model by training and testing on different subsets of data, providing more reliable performance estimates.

K-Fold Cross-Validation:

  1. Split data into K equal parts (folds)
  2. Train on K-1 folds, test on remaining fold
  3. Repeat K times with different test fold each time
  4. Average the K performance scores

Question 19
8 Marks CO5 L3

Explain classification metrics and calculate from confusion matrix.

Answer:

Formulas:

Accuracy = (TP + TN) / (TP + TN + FP + FN) - Overall correctness

Precision = TP / (TP + FP) - Of predicted positives, how many are correct

Recall (Sensitivity) = TP / (TP + FN) - Of actual positives, how many are detected

F1-Score = 2 * (Precision * Recall) / (Precision + Recall) - Harmonic mean

Given Confusion Matrix:

TP = 45, FP = 10, TN = 40, FN = 5

Total = 45 + 10 + 40 + 5 = 100

Calculations:

Accuracy = (45 + 40) / 100 = 85/100 = 0.85 or 85%

Precision = 45 / (45 + 10) = 45/55 = 0.818 or 81.8%

Recall = 45 / (45 + 5) = 45/50 = 0.90 or 90%

F1-Score = 2 * (0.818 * 0.90) / (0.818 + 0.90) = 1.472/1.718 = 0.857 or 85.7%

Most Important for Medical Diagnosis:

Recall (Sensitivity) is most critical for medical diagnosis because:

  • False Negatives (FN) are dangerous - missing a disease can be fatal
  • Better to have false alarms (FP) than miss actual cases
  • High recall ensures maximum disease detection

Calculate regression metrics for customer satisfaction prediction.

Actual Ratings: [3.5, 4.0, 2.8, 4.5, 3.2, 4.8]
Predicted Ratings: [3.2, 4.3, 3.0, 4.2, 3.5, 4.5]

Answer:

(i) Manual Calculations:

Errors (Actual - Predicted): [0.3, -0.3, -0.2, 0.3, -0.3, 0.3]

Absolute Errors: [0.3, 0.3, 0.2, 0.3, 0.3, 0.3]

Squared Errors: [0.09, 0.09, 0.04, 0.09, 0.09, 0.09]

MAE = Sum(|errors|) / n = (0.3+0.3+0.2+0.3+0.3+0.3) / 6 = 1.7/6 = 0.283

MSE = Sum(errors^2) / n = (0.09+0.09+0.04+0.09+0.09+0.09) / 6 = 0.49/6 = 0.082

RMSE = sqrt(MSE) = sqrt(0.082) = 0.286

(ii) Python Verification:

from sklearn.metrics import mean_absolute_error, mean_squared_error
import numpy as np

actual = [3.5, 4.0, 2.8, 4.5, 3.2, 4.8]
predicted = [3.2, 4.3, 3.0, 4.2, 3.5, 4.5]

mae = mean_absolute_error(actual, predicted)
mse = mean_squared_error(actual, predicted)
rmse = np.sqrt(mse)

print(f"MAE: {mae:.4f}")   # 0.2833
print(f"MSE: {mse:.4f}")   # 0.0817
print(f"RMSE: {rmse:.4f}") # 0.2858

(iii) Why RMSE over MSE:

  • RMSE has same units as target variable (ratings), MSE has squared units
  • RMSE = 0.286 means average prediction error is about 0.29 rating points
  • More intuitive for interpretation and communication

(iv) Model Performance Assessment:

Acceptable threshold: 0.5

Our MAE: 0.283, RMSE: 0.286

Yes, the model is performing satisfactorily since both MAE (0.283) and RMSE (0.286) are below the 0.5 threshold.
