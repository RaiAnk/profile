Data Visualization

A Complete Guide to Visualizing Data with Python and Tableau

Overview: The Big Picture

What is Data Visualization?

Data Visualization is the graphical representation of data and information using visual elements like charts, graphs, and maps. It transforms complex datasets into visual formats that are easier to understand, identify patterns, and make informed decisions.

  • "Why visualize?" - Humans process visuals 60,000 times faster than text
  • "What tools?" - Matplotlib (basic), Seaborn (statistical), Tableau (interactive)
  • "When to use?" - Exploratory analysis, presentations, dashboards, reports
The Data Visualization Process: Each step builds on the previous
Think of it Like a Story

Imagine you have to explain your company's sales performance:

  • Without Visualization: "In January sales were 45000, February 52000, March 48000..." - Boring and hard to remember!
  • With Visualization: A line chart instantly shows the trend - growing, declining, or steady. You can spot patterns in seconds!

Data visualization is about making data speak visually so anyone can understand insights quickly!

Three Main Tools Covered:

  • Matplotlib: Python's fundamental plotting library - full control, highly customizable
  • Seaborn: Built on Matplotlib, specializes in statistical visualizations with beautiful defaults
  • Tableau: Professional business intelligence tool for interactive dashboards (no coding required)

Why Data Visualization is Critical

For Data Scientists & Analysts:

  • Quickly spot outliers, patterns, and trends in data
  • Communicate findings effectively to non-technical stakeholders
  • Validate statistical models and assumptions

For Business Decision Makers:

  • Monitor KPIs in real-time through dashboards
  • Make data-driven decisions faster
  • Identify opportunities and risks at a glance

Real-World Impact:

  • John Snow used maps to visualize cholera outbreak and discovered contaminated water pump
  • Florence Nightingale used polar area charts to convince military to improve hospital sanitation
  • Modern dashboards help companies save millions by identifying inefficiencies instantly

Part 1: Matplotlib - Basic Visualization Tools

What is Matplotlib?
Matplotlib is Python's foundational plotting library created by John Hunter in 2003. It provides complete control over every aspect of a figure - from colors and labels to complex multi-plot layouts. Think of it as the "Photoshop" of data visualization - powerful but requires more code.

Analogy: If data visualization were cooking, Matplotlib is like having all the individual ingredients and cooking from scratch. You control everything but need to specify each step. Seaborn (covered later) is like a meal kit - pre-selected ingredients with a recipe for beautiful results faster.

Installation and Setup

# Install Matplotlib (run once in terminal/command prompt) pip install matplotlib # Import in Python script import matplotlib.pyplot as plt import numpy as np import pandas as pd

Level 1: Basic Plotting - Line Charts

Example 1: Simple Line Plot (Temperature Over Week)

Scenario: You have temperature data for one week and want to visualize the trend.

import matplotlib.pyplot as plt # Data: Days and corresponding temperatures days = ['Mon', 'Tue', 'Wed', 'Thu', 'Fri', 'Sat', 'Sun'] temperatures = [22, 24, 19, 25, 27, 26, 23] # Create the plot plt.plot(days, temperatures, marker='o', color='blue', linewidth=2) # Add labels and title plt.xlabel('Day of Week') plt.ylabel('Temperature (°C)') plt.title('Weekly Temperature Trend') # Add grid for better readability plt.grid(True, alpha=0.3) # Display the plot plt.show()
OUTPUT: Temperature Line Chart

Key Parameters Explained:

  • marker='o' - Adds circular dots at data points
  • color - Can be name ('red') or hex code ('#FF5733')
  • linewidth - Thickness of line (default is 1)
  • alpha - Transparency (0=invisible, 1=solid)

Level 2: Multiple Plots and Comparison

Example 2: Comparing Two Datasets (Sales Comparison)

Real-World Scenario: Compare sales performance of two products over 6 months.

import matplotlib.pyplot as plt months = ['Jan', 'Feb', 'Mar', 'Apr', 'May', 'Jun'] product_a_sales = [15000, 18000, 17500, 21000, 23000, 25000] product_b_sales = [12000, 14000, 16000, 15500, 18000, 22000] # Plot both products plt.plot(months, product_a_sales, marker='o', label='Product A', color='#2563eb', linewidth=2) plt.plot(months, product_b_sales, marker='s', label='Product B', color='#dc2626', linewidth=2) plt.xlabel('Month', fontsize=12) plt.ylabel('Sales (USD)', fontsize=12) plt.title('Product Sales Comparison - H1 2024', fontsize=14, fontweight='bold') # Add legend to identify lines plt.legend(loc='upper left') # Format y-axis to show thousands with comma plt.gca().yaxis.set_major_formatter( plt.FuncFormatter(lambda x, p: f'${x/1000:.0f}K') ) plt.grid(True, alpha=0.3) plt.tight_layout() plt.show()
OUTPUT: Sales Comparison Chart
What This Code Does:

Lines 7-10: Creates two separate lines on same plot for comparison

marker='s': Uses square markers for Product B to differentiate from Product A

legend(): Adds a box showing which line represents which product

FuncFormatter: Converts 15000 to "$15K" for cleaner display

tight_layout(): Automatically adjusts spacing to prevent label cutoff

Level 5: Statistical Visualizations

Case Study 1: Analyzing Exam Scores Distribution

Problem: A professor wants to understand if exam scores are normally distributed and identify outliers.

Data: 100 student exam scores

Solution: Create histogram with statistical measures

import matplotlib.pyplot as plt import numpy as np # Generate sample exam scores (normally distributed around 70) np.random.seed(42) scores = np.random.normal(70, 12, 100) scores = np.clip(scores, 0, 100) # Calculate statistics mean_score = np.mean(scores) median_score = np.median(scores) std_dev = np.std(scores) # Create histogram plt.figure(figsize=(10, 6)) n, bins, patches = plt.hist(scores, bins=15, color='skyblue', edgecolor='black', alpha=0.7) # Add vertical lines for mean and median plt.axvline(mean_score, color='red', linestyle='--', linewidth=2, label=f'Mean: {mean_score:.1f}') plt.axvline(median_score, color='green', linestyle='-.', linewidth=2, label=f'Median: {median_score:.1f}') # Add text box with statistics textstr = f'Mean: {mean_score:.2f}\n' textstr += f'Median: {median_score:.2f}\n' textstr += f'Std Dev: {std_dev:.2f}' props = dict(boxstyle='round', facecolor='wheat', alpha=0.5) plt.text(0.75, 0.95, textstr, transform=plt.gca().transAxes, fontsize=10, verticalalignment='top', bbox=props) plt.xlabel('Score', fontsize=12) plt.ylabel('Number of Students', fontsize=12) plt.title('Exam Score Distribution with Statistical Measures', fontsize=14) plt.legend() plt.grid(True, alpha=0.3, axis='y') plt.show() # Print interpretation print(f"\nInterpretation:") print(f"- Scores are approximately normally distributed") print(f"- Mean and median are close, indicating symmetry") print(f"- Std dev of {std_dev:.1f} shows moderate spread")
OUTPUT: Score Distribution Histogram
Key Statistical Measures Used in Data Visualization
Statistical Concepts Applied:

Mean: Average score - sensitive to outliers

Median: Middle value when sorted - resistant to outliers

Standard Deviation: Measures spread - higher value means more variability

Normal Distribution: Bell-shaped curve - most values cluster around mean

If mean and median differ significantly, distribution is skewed (not symmetric)

Part 2: Seaborn - Statistical Data Visualization

What is Seaborn?
Seaborn is a Python visualization library built on top of Matplotlib, designed specifically for statistical graphics. Created by Michael Waskom, it provides beautiful default themes and specializes in visualizing complex datasets with minimal code. It integrates seamlessly with pandas DataFrames.

Matplotlib vs Seaborn: If Matplotlib is manual transmission (full control, more effort), Seaborn is automatic transmission (easier to drive, smart defaults). Seaborn makes common statistical plots simple while Matplotlib handles any custom visualization.

Installation and Setup

# Install Seaborn pip install seaborn # Import libraries import seaborn as sns import matplotlib.pyplot as plt import pandas as pd import numpy as np # Set Seaborn style sns.set_style("whitegrid") # Options: darkgrid, whitegrid, dark, white, ticks sns.set_palette("husl") # Beautiful default colors

Level 1: Relationship Plots

Example 5: Scatter Plot with Regression Line

Scenario: Analyze relationship between study hours and exam scores.

import seaborn as sns import matplotlib.pyplot as plt import pandas as pd import numpy as np # Create sample data np.random.seed(42) study_hours = np.random.uniform(1, 10, 50) # Scores increase with study hours + some randomness exam_scores = 50 + 4 * study_hours + np.random.normal(0, 5, 50) exam_scores = np.clip(exam_scores, 0, 100) # Create DataFrame df = pd.DataFrame({ 'Study Hours': study_hours, 'Exam Score': exam_scores }) # Create scatter plot with regression line plt.figure(figsize=(10, 6)) sns.regplot(data=df, x='Study Hours', y='Exam Score', scatter_kws={'alpha': 0.6, 's': 80}, line_kws={'color': 'red', 'linewidth': 2}) plt.title('Correlation: Study Hours vs Exam Performance', fontsize=14) plt.xlabel('Hours Studied', fontsize=12) plt.ylabel('Exam Score (%)', fontsize=12) # Calculate and display correlation correlation = df['Study Hours'].corr(df['Exam Score']) plt.text(0.05, 0.95, f'Correlation: {correlation:.3f}', transform=plt.gca().transAxes, bbox=dict(boxstyle='round', facecolor='yellow', alpha=0.5), fontsize=11) plt.show() # Interpretation print(f"\nCorrelation coefficient: {correlation:.3f}") if correlation > 0.7: print("Strong positive: More study → Higher scores") elif correlation > 0.3: print("Moderate positive correlation") else: print("Weak or no correlation")
OUTPUT: Correlation Scatter Plot
Understanding Correlation Coefficient Values
Understanding Correlation:

Correlation coefficient (r): Measures strength of linear relationship

  • r = +1: Perfect positive correlation (both increase together)
  • r = 0: No correlation (no relationship)
  • r = -1: Perfect negative correlation (one increases, other decreases)

Important: Correlation does NOT mean causation!

Level 2: Distribution Plots

Example 6: Comparing Distributions Across Categories

Real-World Case: Compare salary distributions across different job roles.

import seaborn as sns import matplotlib.pyplot as plt import pandas as pd import numpy as np # Create sample salary data np.random.seed(42) roles = ['Junior Dev', 'Senior Dev', 'Lead Dev', 'Manager'] data = [] for role, base_salary in zip(roles, [60000, 90000, 120000, 100000]): salaries = np.random.normal(base_salary, 10000, 30) for salary in salaries: data.append({'Role': role, 'Salary': salary}) df = pd.DataFrame(data) # Create violin plot (shows distribution shape) plt.figure(figsize=(12, 6)) sns.violinplot(data=df, x='Role', y='Salary', palette='muted') # Add individual points sns.swarmplot(data=df, x='Role', y='Salary', color='black', alpha=0.3, size=3) plt.title('Salary Distribution by Role', fontsize=14, fontweight='bold') plt.ylabel('Annual Salary (USD)', fontsize=12) plt.xlabel('Job Role', fontsize=12) # Format y-axis plt.gca().yaxis.set_major_formatter( plt.FuncFormatter(lambda x, p: f'${x/1000:.0f}K') ) plt.show() # Calculate statistics for each role print("\nSalary Statistics by Role:") print(df.groupby('Role')['Salary'].describe())
OUTPUT: Salary Distribution Violin Plot

Violin Plot vs Box Plot:

  • Violin Plot: Shows full distribution shape (density), can reveal multiple peaks
  • Box Plot: Shows quartiles and outliers, simpler but less detailed
  • Swarm Plot: Shows every individual point without overlap
  • When to use: Violin for distributions, Box for quick summaries, Swarm for small datasets

Level 4: Heatmaps for Correlation Matrices

Example 7: Feature Correlation Analysis

Use Case: Analyzing relationships between multiple variables in a dataset.

import seaborn as sns import matplotlib.pyplot as plt import pandas as pd import numpy as np # Create sample housing data np.random.seed(42) n = 100 df = pd.DataFrame({ 'Size_sqft': np.random.uniform(800, 3000, n), 'Bedrooms': np.random.randint(1, 6, n), 'Age_years': np.random.uniform(0, 50, n), 'Distance_to_city_km': np.random.uniform(1, 50, n) }) # Price depends on size (+), age (-), distance (-) df['Price_1000s'] = ( 150 + 0.2 * df['Size_sqft'] + 20 * df['Bedrooms'] - 2 * df['Age_years'] - 3 * df['Distance_to_city_km'] + np.random.normal(0, 20, n) ) # Calculate correlation matrix correlation_matrix = df.corr() # Create heatmap plt.figure(figsize=(10, 8)) sns.heatmap(correlation_matrix, annot=True, cmap='coolwarm', center=0, square=True, linewidths=1, cbar_kws={"shrink": 0.8}, fmt='.2f') plt.title('Housing Features Correlation Matrix', fontsize=14, fontweight='bold') plt.tight_layout() plt.show() # Identify strongest correlations with price price_corr = correlation_matrix['Price_1000s'].drop('Price_1000s') price_corr = price_corr.sort_values(ascending=False) print("\nFactors affecting price (sorted by correlation):") print(price_corr)
OUTPUT: Correlation Heatmap
Reading a Correlation Heatmap:

Colors: Red = positive correlation, Blue = negative correlation, White = no correlation

Diagonal: Always 1.0 (perfect correlation with itself)

Symmetric: Top-right mirrors bottom-left

Numbers: Correlation coefficients from -1 to +1

Business Insight: Strong correlations help predict one variable from others (predictive modeling)

    Combining Visualization with Machine Learning

    Case Study 5: Customer Segmentation with Clustering

    Business Goal: E-commerce company wants to identify distinct customer groups for targeted marketing

    Approach: Use K-Means clustering algorithm + visualize segments

    Data: Customer purchase history - Total Spent, Frequency, Recency

    import numpy as np import pandas as pd import matplotlib.pyplot as plt import seaborn as sns from sklearn.cluster import KMeans from sklearn.preprocessing import StandardScaler # Generate sample customer data np.random.seed(42) n_customers = 200 data = pd.DataFrame({ 'Total_Spent': np.random.exponential(500, n_customers), 'Purchase_Frequency': np.random.poisson(5, n_customers), 'Days_Since_Last_Purchase': np.random.gamma(2, 20, n_customers) }) # Standardize features (important for clustering) scaler = StandardScaler() data_scaled = scaler.fit_transform(data) # Apply K-Means clustering (3 segments) kmeans = KMeans(n_clusters=3, random_state=42, n_init=10) data['Segment'] = kmeans.fit_predict(data_scaled) # Assign segment names based on characteristics segment_names = { 0: 'High Value', 1: 'At Risk', 2: 'Occasional' } data['Segment_Name'] = data['Segment'].map(segment_names) # Visualization: Scatter plot fig, (ax1, ax2) = plt.subplots(1, 2, figsize=(14, 6)) # Subplot 1: Spending vs Frequency for segment in data['Segment_Name'].unique(): subset = data[data['Segment_Name'] == segment] ax1.scatter(subset['Total_Spent'], subset['Purchase_Frequency'], label=segment, alpha=0.6, s=100) ax1.set_xlabel('Total Spent ($)', fontsize=12) ax1.set_ylabel('Purchase Frequency', fontsize=12) ax1.set_title('Customer Segments - Spending Behavior', fontsize=14, fontweight='bold') ax1.legend() ax1.grid(True, alpha=0.3) # Subplot 2: Box plot comparison data.boxplot(column='Total_Spent', by='Segment_Name', ax=ax2) ax2.set_title('Spending Distribution by Segment', fontsize=14, fontweight='bold') ax2.set_xlabel('Segment', fontsize=12) ax2.set_ylabel('Total Spent ($)', fontsize=12) plt.tight_layout() plt.show() # Statistical summary of segments print("\n=== Customer Segment Profiles ===\n") for segment in data['Segment_Name'].unique(): subset = data[data['Segment_Name'] == segment] print(f"\n{segment} Customers (n={len(subset)}):") print(f" Average Spending: ${subset['Total_Spent'].mean():.2f}") print(f" Average Frequency: {subset['Purchase_Frequency'].mean():.1f}") # Business recommendations print("\n=== Marketing Recommendations ===\n") print("High Value: Send VIP offers, loyalty rewards") print("At Risk: Re-engagement campaigns, discounts") print("Occasional: Nurture with content, promotions")
    OUTPUT: Customer Segmentation Clustering
    Algorithm Explanation: K-Means Clustering

    How it works:

    1. Randomly place K "centroid" points (K=3 for 3 segments)
    2. Assign each customer to nearest centroid
    3. Move centroids to average position of assigned customers
    4. Repeat steps 2-3 until centroids stop moving

    Why standardize? Variables have different scales ($500 vs 5 purchases). StandardScaler makes them comparable.

    Case Study 6: Sales Forecasting for Inventory Planning

    Problem: Retail company needs to predict next 3 months sales to optimize inventory

    Data: 2 years of historical daily sales

    Method: Moving average + seasonal decomposition

    import pandas as pd import numpy as np import matplotlib.pyplot as plt from statsmodels.tsa.seasonal import seasonal_decompose # Generate sample sales data with trend and seasonality date_range = pd.date_range(start='2022-01-01', end='2023-12-31', freq='D') n_days = len(date_range) # Components: Trend + Seasonality + Random trend = np.linspace(1000, 1500, n_days) seasonal = 200 * np.sin(np.linspace(0, 4*np.pi, n_days)) random_noise = np.random.normal(0, 50, n_days) sales = trend + seasonal + random_noise df = pd.DataFrame({ 'Date': date_range, 'Sales': sales }) df.set_index('Date', inplace=True) # Decompose time series decomposition = seasonal_decompose(df['Sales'], model='additive', period=365) # Calculate moving averages (different windows) df['MA_7'] = df['Sales'].rolling(window=7).mean() df['MA_30'] = df['Sales'].rolling(window=30).mean() # Create comprehensive visualization fig, axes = plt.subplots(4, 1, figsize=(14, 12)) # Plot 1: Original with moving averages axes[0].plot(df.index, df['Sales'], label='Actual Sales', alpha=0.5, color='gray') axes[0].plot(df.index, df['MA_7'], label='7-Day MA', color='blue') axes[0].plot(df.index, df['MA_30'], label='30-Day MA', color='red', linewidth=2) axes[0].set_title('Sales with Moving Averages', fontsize=12, fontweight='bold') axes[0].set_ylabel('Sales ($)', fontsize=11) axes[0].legend() axes[0].grid(True, alpha=0.3) # Plot 2: Trend component axes[1].plot(decomposition.trend, color='green', linewidth=2) axes[1].set_title('Trend Component (Overall Direction)', fontsize=12, fontweight='bold') axes[1].set_ylabel('Trend', fontsize=11) axes[1].grid(True, alpha=0.3) # Plot 3: Seasonal component axes[2].plot(decomposition.seasonal, color='orange', linewidth=2) axes[2].set_title('Seasonal Component (Repeating Pattern)', fontsize=12, fontweight='bold') axes[2].set_ylabel('Seasonality', fontsize=11) axes[2].grid(True, alpha=0.3) # Plot 4: Residual component axes[3].plot(decomposition.resid, color='purple', alpha=0.7) axes[3].set_title('Residual Component (Random Noise)', fontsize=12, fontweight='bold') axes[3].set_ylabel('Residual', fontsize=11) axes[3].set_xlabel('Date', fontsize=11) axes[3].grid(True, alpha=0.3) plt.tight_layout() plt.show()
    OUTPUT: Time Series Decomposition
    Understanding Time Series Components:

    Trend: Long-term increase or decrease (e.g., business growing)

    Seasonality: Regular patterns repeating over fixed period (e.g., holiday spikes)

    Noise: Random variations (e.g., unpredictable events)

    Moving Average: Smooths data by averaging over window - reduces noise

    Forecasting = Trend + Expected Seasonality

    Case Study 7: Website Button Color A/B Test

    Experiment: Test if green button performs better than blue button

    Metrics: Click-through rate (CTR)

    Statistical Question: Is difference statistically significant?

    import numpy as np import pandas as pd import matplotlib.pyplot as plt import seaborn as sns from scipy import stats # Experiment data control = { 'variant': 'Control (Blue)', 'visitors': 10000, 'clicks': 850 } treatment = { 'variant': 'Treatment (Green)', 'visitors': 10000, 'clicks': 920 } # Calculate metrics control['ctr'] = control['clicks'] / control['visitors'] * 100 treatment['ctr'] = treatment['clicks'] / treatment['visitors'] * 100 improvement = ((treatment['ctr'] - control['ctr']) / control['ctr']) * 100 # Statistical significance test (Chi-square) contingency_table = np.array([ [control['clicks'], control['visitors'] - control['clicks']], [treatment['clicks'], treatment['visitors'] - treatment['clicks']] ]) chi2, p_value, dof, expected = stats.chi2_contingency(contingency_table) # Visualization fig, axes = plt.subplots(1, 2, figsize=(15, 6)) # Plot 1: CTR Comparison variants = [control['variant'], treatment['variant']] ctrs = [control['ctr'], treatment['ctr']] colors = ['#3b82f6', '#10b981'] bars = axes[0].bar(variants, ctrs, color=colors, edgecolor='black', linewidth=2) axes[0].set_ylabel('Click-Through Rate (%)', fontsize=12) axes[0].set_title('A/B Test Results', fontsize=14, fontweight='bold') axes[0].set_ylim(0, 10) # Add value labels for bar, ctr in zip(bars, ctrs): axes[0].text(bar.get_x() + bar.get_width()/2, ctr + 0.2, f'{ctr:.2f}%', ha='center', fontsize=12, fontweight='bold') # Plot 2: Statistical summary axes[1].axis('off') result_text = f""" A/B Test Summary ───────────────────────── Control CTR: {control['ctr']:.2f}% Treatment CTR: {treatment['ctr']:.2f}% Improvement: {improvement:+.2f}% Statistical Significance: P-value: {p_value:.4f} Result: {'✓ SIGNIFICANT' if p_value < 0.05 else '✗ NOT SIGNIFICANT'} Recommendation: {'Implement green button!' if p_value < 0.05 and improvement > 0 else 'Keep testing'} """ axes[1].text(0.1, 0.5, result_text, fontsize=11, verticalalignment='center', bbox=dict(boxstyle='round', facecolor='lightgreen' if p_value < 0.05 else 'lightyellow', alpha=0.5), family='monospace') plt.tight_layout() plt.show() # Detailed report print(f"\n=== Detailed Statistical Report ===") print(f"Control: {control['clicks']:,} clicks / {control['visitors']:,}") print(f"Treatment: {treatment['clicks']:,} clicks / {treatment['visitors']:,}") print(f"P-value: {p_value:.4f}") if p_value < 0.05: print(f"✓ Statistically significant (p < 0.05)") else: print(f"✗ NOT significant - need more data")
    OUTPUT: A/B Test Results
    Understanding P-Value:

    P-value: Probability that difference occurred by random chance

    • p < 0.05: Less than 5% chance it's random → SIGNIFICANT
    • p >= 0.05: More than 5% chance it's random → NOT significant

    Business Rule: Only implement changes that are statistically significant to avoid false positives

    Summary and Key Takeaways

    Matplotlib - Foundation of Python Visualization

    • Complete control over every visual element
    • Essential functions: plot(), bar(), scatter(), hist(), subplot()
    • Best for: Custom visualizations, publication-quality figures
    • Key concept: Figure contains Axes, Axes contain plots
    • Remember: plt.show() displays, plt.savefig() saves

    Seaborn - Statistical Visualization Made Easy

    • Built on Matplotlib with beautiful defaults
    • Specializes in: Distribution plots, correlation, categorical data
    • Key functions: regplot(), violinplot(), heatmap(), pairplot()
    • Works seamlessly with pandas DataFrames
    • Best for: Exploratory data analysis, statistical relationships

    Tableau - Business Intelligence Powerhouse

    • No coding required - drag and drop interface
    • Core concepts: Dimensions (categorical), Measures (numerical)
    • Interactive by default - filters, drill-downs, tooltips
    • Dashboard capabilities: Multiple charts, unified filtering, stories
    • Best for: Executive dashboards, business presentations, real-time monitoring

    Statistical Measures in Visualization (CO4)

    • Central Tendency: Mean (average), Median (middle), Mode (most common)
    • Spread: Standard Deviation, Variance, Range, IQR
    • Relationships: Correlation coefficient, R-squared
    • Distribution: Skewness, Kurtosis, Normality tests
    • Inference: P-values, Confidence intervals, Hypothesis testing

    Common Visualization Mistakes to Avoid:

    • Using pie charts for more than 5 categories
    • Not starting bar chart y-axis at zero (misleading)
    • Too many colors making chart confusing
    • Cluttered dashboards with no clear focus
    • 3D charts when 2D would be clearer
    • Missing axis labels or unclear titles

    Choosing the Right Chart Type:

    • Comparison: Bar chart (categories), Line chart (time)
    • Distribution: Histogram (one variable), Box plot (multiple groups)
    • Relationship: Scatter plot (correlation), Heatmap (multiple variables)
    • Composition: Pie chart (< 5 parts), Stacked bar (over time)
    • Geographic: Filled map (regional), Bubble map (cities)

    The Golden Rule of Visualization

    A great visualization tells a story without words - it should be self-explanatory and actionable

    The Process: Data → Insight → Visual → Action

    If your audience needs 5 minutes to understand your chart, redesign it!
